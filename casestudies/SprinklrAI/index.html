Sprinklr AI+ Design System




Project Type

Design System

Duration

4 Months

January 2024 - May 2024

My Role

Interaction Design for modals and chat components

Visual Design ensuring brand consistency and alignment with Sprinklr’s atomic components

Prototyping for subtle and engaging loading states

Contribution

I contributed to Sprinklr’s AI+ Design System, focusing on creating cohesive Modals, Widgets, Chat Components, and Loading Animations. My goal was to ensure consistency, usability, and a branded experience for Sprinklr’s AI features.

Sprinklr’s AI components lacked a unified design approach for modals and chat interactions, causing inconsistent user experiences .

Research & Requirements Gathering

I reviewed existing AI screens, components, user feedback, and Sprinklr brand guidelines to identify gaps. Setting up these guidelines will help improve brand consistency, reduce user confusion, and streamline workflows. By creating modular and reusable UI elements (like modals, widgets, and chat components), the design system minimizes the need for redundant design work. This accelerates development cycles, allowing Sprinklr to bring new features and products to market faster.

For this project, I worked closely with Sprinklr Design System manager, a developer and a product manager in order to create consistent design tokens (colors, typography, spacing) to reduce the back-and-forth between design and development teams. Faster iteration and lower maintenance costs directly impact overall project budgets and profitability.

All components follow Sprinklr's unified atomic design system , ensuring consistency, maintainability, and scalability.

Design Focus:

  • Basic modal layout with a clear title, message body, and primary/secondary buttons.

  • A straightforward structure to test basic interactions and hierarchy.

Visual Elements:

  • Simple color usage with placeholder tones.

  • Basic typography and standard button shapes.

  • Minimal use of shadows or borders.

Feedback & Learnings:

  • The initial modal was perceived as too static and lacked clear visual hierarchy.

  • Early user feedback highlighted the need for enhanced accessibility and clearer CTAs.

Design Focus:

  • Introduced improved typography for better readability.

  • Refined layout spacing to distinguish title, content, and actions more clearly.

  • Added subtle shadow effects and rounded corners to enhance the visual appeal.

Visual Elements:

  • Updated color palette integrating Sprinklr’s AI brand hues.

  • Introduction of micro-interactions (e.g., button hover states, fade-in for the modal).

  • A more defined loading state animation to communicate progress.

Feedback & Learnings:

  • Users appreciated the improved clarity and visual cues.

  • Adjustments were made based on developer feedback for smoother integration with the atomic components system.

Design Focus:

  • Final fine-tuning of spacing, contrast, and accessibility.

  • Optimized for responsiveness and consistency across devices

  • Polished animations ensuring subtle yet effective loading feedback.

Visual Elements:

  • Consistent use of the Sprinklr AI color scheme and typography.

  • Clear iconography and callouts for important actions.

  • Refined modal transitions that enhance user engagement without overwhelming the experience.

Outcome:

  • A modern, accessible modal that balances aesthetics with functionality and meets design system standards. The consistent design tokens (colors, typography, spacing) help to reduce the back-and-forth between design and development teams. Faster iteration and lower maintenance costs directly impact overall project budgets and profitability.

Use Cases:

  • Popup for when users don’t need to navigate away.

  • If there is a footer, use popup pattern.

  • If it is already a modal, use popup pattern.

Use Cases:

  • Lightbox for when users have to make a decision and the interface is blocked out.

  • Could have multi-steps.
